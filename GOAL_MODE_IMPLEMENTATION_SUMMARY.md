# Goal模式实现总结

## 一、实现方式（自然语言描述）

### 核心思想

Goal模式是一个**自动重试机制**，用于确保AI生成的内容符合预期。它的核心思想是：

> **定义目标 → 执行 → 验证 → 重试 → 直到达标**

### 实现架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Goal模式架构                           │
├─────────────────────────────────────────────────────────────┤
│  1. Goal定义层                                              │
│     - 目标类型（转表格/转图表/转流程图/添加图片）            │
│     - 验证规则（字段存在性、数据结构、质量阈值）            │
│     - 最大尝试次数限制（默认99次）                          │
├─────────────────────────────────────────────────────────────┤
│  2. Prompt生成层                                            │
│     - 根据目标类型生成基础prompt                            │
│     - 包含明确的字段名和数据结构示例                        │
│     - 根据尝试次数动态调整约束强度                          │
├─────────────────────────────────────────────────────────────┤
│  3. 执行验证层                                              │
│     - 调用AI生成响应                                        │
│     - 解析响应提取actions                                   │
│     - 应用actions到测试数据                                 │
│     - 执行质量检查（字段校验、结构验证）                    │
├─────────────────────────────────────────────────────────────┤
│  4. 反馈重试层                                              │
│     - 如果未达标，提取失败原因                              │
│     - 将失败原因添加到下次prompt中                          │
│     - 强化约束，补充示例                                    │
│     - 重新执行，直到达标或超限                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键设计点

**1. 目标定义（Goal）**
- 每个目标都有明确的类型（如TEXT_TO_TABLE）
- 定义验证规则（如table_data必须有columns和rows字段）
- 设置最大尝试次数（默认99次）

**2. Prompt生成策略**
- **首次尝试**：提供完整的字段名规范和示例格式
- **重试时**：添加上次失败的反馈，强化约束
- **动态调整**：根据尝试次数增加强调语气

**3. 验证机制**
- **字段存在性**：检查必需字段是否存在
- **键名一致性**：检查字段名是否符合规范
- **数据类型合规性**：检查数据类型是否正确
- **质量阈值**：设置通过阈值（如80%以上算成功）

**4. 反馈循环**
- 提取失败原因（如"缺少columns字段"）
- 将失败原因添加到下次prompt中
- 让AI知道上次哪里出错，避免重复错误

### 执行流程

```
开始
  ↓
设置目标（Goal）
  ↓
生成prompt（包含字段规范和示例）
  ↓
调用AI生成响应
  ↓
解析响应提取actions
  ↓
应用actions到数据
  ↓
执行质量检查
  ↓
达标？ ─是→ 返回成功
  ↓否
达到最大尝试次数（99次）？ ─是→ 返回失败
  ↓否
提取失败原因
  ↓
优化prompt（添加反馈、强化约束）
  ↓
重新执行（循环）
```

### 核心优势

**1. 自动化**
- 无需人工干预
- 自动重试直到达标

**2. 智能化**
- 根据失败原因调整策略
- 动态优化prompt

**3. 可控性**
- 设置最大尝试次数（99次）
- 用户可强制终止
- UI开关可选择是否开启

**4. 可追溯**
- 记录每次尝试的结果
- 便于分析和优化

## 四、使用方式

### 1. 开启Goal模式

在共创Widget的toolbar中，点击"🎯 Goal模式"复选框：
- ✅ 勾选：开启Goal模式
- ☐ 取消：关闭Goal模式

### 2. Goal模式效果

**开启后**：
- 转图表、图片、流程图等操作会自动启用Goal模式
- AI会自动重试直到生成高质量产物
- 最多尝试99次
- 状态指示器显示"🎯 已开启"

**关闭后**：
- 使用普通模式
- 只尝试1次
- 不保证质量

### 3. 适用场景

**推荐开启**：
- 转图表操作（文本→表格/图表/流程图）
- 图片操作（添加配图描述）
- 对质量要求高的场景

**可以关闭**：
- 简单的内容修改（优化标题、添加要点）
- 对响应速度要求高的场景
