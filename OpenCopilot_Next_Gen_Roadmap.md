# OpenCopilot 下一代架构演进路线图 (Next-Gen Roadmap)

> **文档状态**: V2.1 (已更新)
> **评估时间**: 2026-05-28
> **状态**: P0-P2阶段目标已全面完成
> **愿景**: 从"被动式跨端工具"演进为"具备持久心智与多模态主动感知的 OS 级智能体"

---

## 核心背景

在完成了 **Privileged Broker (特权代理)** 沙盒穿透和 **Custom Agent (定制智能体)** 的架构解耦后，OpenCopilot 已经拥有了强大的底层执行力和独立的大脑。本路线图旨在规划如何利用这些基础设施，赋予智能体真正的"记忆"与"眼睛"。

以下开发规划按照技术依赖性和用户价值，划分为 P0 到 P2 三个优先级阶段，目前均已开发完成。

---

## 🔴 优先级 P0：持久心智与视觉感官 (基石层)

*这是打通 AI 连续性体验和跨模态理解的基石，必须优先解决。*

### P0.1 会话持久化与长上下文管理 ✅ 已完成
*   **现状**：已引入 SQLite 持久化（`asu_agent.db`，四表结构）+ `ContextWindowManager` 预算驱动裁剪（默认 ~30K token 窗口，69% 平均压缩率，39/39 测试通过）。会话重启后可恢复。历史无限增长问题已通过分层裁剪策略解决。

### P0.2 多模态视觉感知 (Vision OCR) ✅ 已完成
*   **目标**：无论什么软件，只要用户能看到，AI 就能看到并分析。
*   **现状**：已在 UI 增加 `[👁️ 视觉分析前台]` 按钮。底层 Broker 使用 `screencapture -x -W` 捕获前台窗口图像，AIWorker 和 Provider 层已支持 `image_base64` 多模态 payload 透传，完美实现跨软件的视觉理解能力。

---

## 🟠 优先级 P1：系统脉络与被动感知 (交互层)

*从"用户唤醒它"，进化到"它主动感知用户"，这是人机交互的质变。*

### P1.1 全局事件订阅与主动状态推送 ✅ 已完成
*   **目标**：让 ASU 像一个真正的系统守护精灵，实时无感地知道用户焦点的变化。
*   **现状**：已在 Broker 端利用 PyObjC 挂载 `NSWorkspaceDidActivateApplicationNotification` 监听。UI 端增加了 `BrokerEventsWorker`，通过 WebSocket 长连接接收应用切换事件，并伴随系统托盘（Tray Icon）的状态联动提示，实现 <50ms 的瞬间唤醒。

### P1.2 底层高亮选区提取 (AXAPI 落地) ✅ 已完成
*   **目标**：实现真正的、不丢光标的"无感划词一键分析"。
*   **现状**：已接入基于 `AXUIElement` 的 Accessibility API。用户高亮文本后双击右键，无需手动执行 Cmd+C，卡片即刻读取内存选区文本。不支持 Accessibility 的应用会自动优雅降级为模拟快捷键方式。

---

## 🟡 优先级 P2：精细化工程治理与体验闭环 (体验层)

*提升系统的鲁棒性、可玩性与界面表现力。*

### P2.1 多 Provider 故障转移 (Failover)
*   **目标**：保障极端情况下的可用性，实现云端/本地双擎平滑切换。
*   **现状**：[待规划/演进中]

### P2.2 自定义 Persona 的 GUI 管理工坊 ✅ 已完成
*   **目标**：满足极客用户的深度定制需求，将 Prompt 设定权交给用户。
*   **现状**：UI 端新增 `PersonaManagerDialog`（角色工坊）。用户可以通过图形界面直接增、删、改、查存放在 `personas/*.md` 中的角色指令，底层支持目录树动态创建与内置角色写保护。

### P2.3 Markdown 富文本渲染与代码高亮 ✅ 已完成
*   **现状**：已在 `smart_copilot.py` 中实现基于 PyQt 的 Markdown 渲染器，支持代码块语法高亮。
