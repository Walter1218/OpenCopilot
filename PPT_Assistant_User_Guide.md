# PPT 助手使用手册

> **版本**：v1.0 | **日期**：2026-05-31 | **状态**：功能完整，持续迭代中

---

## 一、功能概述

PPT 助手是 OpenCopilot 的核心功能模块，提供 **AI 驱动的 PPT 共创编辑器**，支持：

- **领域自适应生成**：AI 自动识别内容领域（计算机/AI/商业/产品/教育/数据分析），采用对应的专业结构模板
- 智能内容生成：AI 自动分析文本，生成结构化大纲
- 可视化编辑：三面板布局（原文 + 大纲 + 预览）
- 对话式修改：用自然语言指令调整幻灯片内容
- 多种布局：支持图文混排、三栏对比等多种版式
- 实时预览：所见即所得，修改即时生效

## 三、创建 PPT 的三种方式

### 方式1：从文本创建（推荐）

**适用场景**：已有内容，快速转化为 PPT

**操作步骤**：
1. 点击「📝 从文本创建 PPT」按钮
2. 在弹出的输入框中输入或粘贴内容
3. 点击「OK」确认
4. AI 自动生成大纲（约 3-5 秒）
5. 进入共创编辑器

**输入格式建议**：
```
一、项目背景
- 市场需求分析
- 竞争格局
- 项目目标

二、技术方案
- 架构设计
- 核心技术
- 创新点

三、实施计划
- 时间节点
- 里程碑
- 资源需求
```

### 方式2：对话式创建

**适用场景**：在对话过程中顺手创建 PPT

**操作步骤**：
1. 切换到「💬 连续对话」Tab
2. 输入指令，例如：
   - "把这段内容做成 PPT"
   - "帮我做一个关于 AI 发展的 PPT"
   - "将以下文档整理成演示文稿"
3. AI 生成内容后，右下角出现「💾 导出为 PPT」按钮
4. 点击按钮进入共创编辑器

**示例指令**：
```
# 基础指令
"把这段内容做成 PPT"
"帮我做一个产品介绍 PPT"

# 指定内容
"做一个关于 OpenCopilot 架构的 PPT，包含系统架构、核心模块、技术栈"

# 处理文档
"把这份会议纪要整理成 PPT"
"将这个 README 转换成演示文稿"
```

### 方式3：拖拽文档

**适用场景**：已有文档文件，直接导入

**操作步骤**：
1. 将文档文件（.txt/.md/.pdf）拖拽到 Smart Copilot 窗口
2. 发送指令："把这份文档做成 PPT"
3. AI 分析文档并生成大纲
4. 点击「💾 导出为 PPT」进入编辑器

**支持的文件格式**：
- `.txt` - 纯文本文件
- `.md` - Markdown 文件
- `.pdf` - PDF 文档

## 四、AI 领域自适应能力

PPT 助手的 AI 会自动识别输入内容的专业领域，并采用对应的结构模板生成大纲。

### 支持的领域

| 领域 | 结构模板 | 适用内容 |
|------|----------|----------|
| **计算机/软件/工程** | 概述→架构→模块→实现→性能→部署 | 技术文档、系统设计、工程方案 |
| **人工智能/ML** | 背景→问题→数据→模型→实验→对比→方向 | 论文、研究报告、算法介绍 |
| **商业/管理/市场** | 市场→痛点→方案→模式→优势→预测→行动 | 商业计划、市场分析、竞品报告 |
| **产品/设计** | 需求→定位→功能→交互→架构→迭代 | 产品介绍、设计方案、需求文档 |
| **教育/培训** | 目标→框架→概念→案例→练习→回顾 | 培训课件、教学材料、知识分享 |
| **数据分析/报告** | 概览→指标→趋势→洞察→问题→建议 | 数据报告、分析文档、工作总结 |

### 工作原理

```
用户输入内容
    ↓
AI 分析：属于哪个领域？
    ↓
选择对应的结构模板和标题风格
    ↓
生成专业化的 PPT 大纲
```

**示例**：
- 输入"微服务架构设计" → 自动采用技术文档结构
- 输入"Q3 市场分析报告" → 自动采用商业分析结构
- 输入"Transformer 模型原理" → 自动采用学术报告结构

## 五、共创编辑器详解

### 4.1 界面布局

```
┌─────────────────────────────────────────────────────────┐
│  工具栏：主题切换 | 全屏预览 | 导出 | 快捷键帮助        │
├──────────┬──────────────────┬───────────────────────────┤
│          │                  │                           │
│  原文    │    大纲编辑      │       PPT 预览            │
│  面板    │    面板          │       面板                │
│          │                  │                           │
│  - 原始  │  - 幻灯片列表    │  - 实时渲染               │
│    文本  │  - 编辑表单      │  - 所见即所得             │
│  - 高亮  │  - 布局选择      │  - 全屏预览               │
│    标记  │  - 要点管理      │                           │
│          │                  │                           │
├──────────┴──────────────────┴───────────────────────────┤
│  AI 对话框：输入自然语言指令修改幻灯片                    │
├─────────────────────────────────────────────────────────┤
│  状态栏：幻灯片数 | 要点数 | 原文提炼率                  │
└─────────────────────────────────────────────────────────┘
```

### 4.2 原文面板功能

| 功能 | 操作方式 | 说明 |
|------|----------|------|
| 查看原文 | 滚动浏览 | 显示导入的原始文本 |
| 选中文本 | 鼠标拖拽 | 选中后自动高亮对应幻灯片 |
| 创建幻灯片 | 选中 → 右键 → 创建幻灯片 | 将选中内容添加为新幻灯片 |
| 定位关联 | 点击文本 | 自动跳转到对应幻灯片 |

### 4.3 大纲面板功能

| 功能 | 操作方式 | 说明 |
|------|----------|------|
| 选择幻灯片 | 点击列表项 | 切换到该幻灯片进行编辑 |
| 修改标题 | 编辑标题字段 | 直接输入新标题 |
| 修改副标题 | 编辑副标题字段 | 仅标题页有此字段 |
| 选择布局 | 下拉菜单 | 选择合适的版式 |
| 添加要点 | 点击「+ 添加要点」 | 在末尾添加新要点 |
| 编辑要点 | 直接编辑文本 | 修改要点内容 |
| 删除要点 | 点击「×」按钮 | 删除该要点 |
| 调整顺序 | 拖拽列表项 | 上下拖拽调整顺序 |
| 删除幻灯片 | 右键 → 删除 | 删除整页幻灯片 |

### 4.4 预览面板功能

| 功能 | 操作方式 | 说明 |
|------|----------|------|
| 实时预览 | 自动更新 | 修改大纲后即时预览效果 |
| 切换幻灯片 | 点击缩略图 | 查看不同页面 |
| 全屏预览 | 点击全屏按钮 | 大屏查看最终效果 |
| 导出预览 | 点击导出按钮 | 预览导出效果 |

### 4.5 AI 对话框功能

在编辑器底部的输入框中，输入自然语言指令修改幻灯片：

#### 修改标题
```
"把第2页的标题改成'技术架构'"
"修改第1页副标题为'AI 驱动的未来'"
```

#### 添加内容
```
"在第3页添加一个要点：性能优化"
"给第2页增加三个技术优势"
```

#### 删除内容
```
"删除最后一页"
"移除第3页的第二个要点"
```

#### 修改布局
```
"把第4页改成三栏布局"
"第2页使用图文混排"
```

#### 重写内容
```
"重写第2页，重点突出技术优势"
"把第3页的内容精简一下"
```

#### 调整结构
```
"在第2页和第3页之间插入一页"
"把第4页移到最后"
```

## 六、布局类型详解

### 5.1 center - 标题居中

**适用场景**：封面页、章节页、结束页

**效果**：
```
┌─────────────────────────┐
│                         │
│                         │
│       演示标题          │
│       副标题            │
│                         │
│                         │
└─────────────────────────┘
```

### 5.2 text_only - 纯文本

**适用场景**：内容页、要点页、说明页

**效果**：
```
┌─────────────────────────┐
│  页面标题               │
├─────────────────────────┤
│  • 要点一               │
│  • 要点二               │
│  • 要点三               │
│                         │
└─────────────────────────┘
```

### 5.3 image_right - 图文混排

**适用场景**：产品介绍、案例展示、功能说明

**效果**：
```
┌─────────────────────────┐
│  页面标题               │
├────────────────┬────────┤
│  • 要点一      │        │
│  • 要点二      │  图片  │
│  • 要点三      │        │
└────────────────┴────────┘
```

### 5.4 three_columns - 三栏对比

**适用场景**：对比分析、方案比较、特性展示

**效果**：
```
┌─────────────────────────┐
│       页面标题          │

│  标题1  │  标题2  │标题3│

│  • 要点 │  • 要点 │•要点│

```

## 九、导出 PPT

### 8.1 导出步骤

1. 调整满意后，点击底部「💾 确认并导出 PPT」按钮
2. 选择保存位置（默认桌面）
3. 输入文件名（默认 `generated_presentation.pptx`）
4. 点击「保存」
5. macOS 自动用 Keynote/PowerPoint 打开

### 8.2 导出位置

- **默认路径**：`~/Desktop/generated_presentation.pptx`
- **可自定义**：导出时可选择任意位置

### 8.3 导出格式

- **格式**：`.pptx`（Microsoft PowerPoint 格式）
- **兼容性**：支持 Keynote、PowerPoint、WPS、LibreOffice

## 十、常见问题

### Q1：AI 生成的大纲不满意怎么办？

**解决方案**：
1. 在共创编辑器中手动修改
2. 使用 AI 对话框指令调整
3. 重新生成：返回重新输入内容

### Q2：支持哪些输入格式？

**支持格式**：
- 纯文本（直接输入/粘贴）
- Markdown 格式
- `.txt` 文件
- `.md` 文件
- `.pdf` 文件

### Q3：生成的 PPT 在哪？

**默认位置**：桌面，文件名格式 `generated_presentation.pptx`

### Q4：如何修改主题/样式？

**操作方式**：编辑器右上角下拉菜单选择主题

### Q5：AI 对话框不响应怎么办？

**解决方案**：
1. 检查 Agent 服务是否运行：`curl http://127.0.0.1:18888/health`
2. 重新启动服务：`python start_broker_with_kg.py`
3. 检查网络连接

### Q6：导出的 PPT 打不开怎么办？

**解决方案**：
1. 确认安装了 Keynote 或 PowerPoint
2. 尝试用 WPS 打开
3. 检查文件是否完整（文件大小应 > 10KB）

## 十一、使用技巧

### 10.1 内容组织技巧

- **结构清晰**：使用一、二、三... 或 1、2、3... 分章节
- **要点精简**：每页 3-5 个要点，每个要点一行
- **层次分明**：使用缩进表示层级关系

### 10.2 AI 对话技巧

- **明确指定**："把**第2页**的标题改成..."
- **具体描述**："添加一个要点：**性能优化，响应时间 < 100ms**"
- **渐进修改**：一次只修改一个地方，确认后再继续

### 10.3 效率提升技巧

- **使用快捷键**：`Cmd+S` 保存，`↑↓` 切换页面
- **批量操作**：先完成所有内容编辑，再调整布局
- **及时保存**：编辑过程中定期导出备份
