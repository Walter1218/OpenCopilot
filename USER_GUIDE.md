# OpenCopilot 用户说明书

> **版本**：v2.1 | **适用平台**：macOS 15+ | **最后更新**：2026-05-28  
> **状态**：P0-P2 阶段（主动感知、多模态、无感划词、Persona 工作坊）已全面完成

---

## 一、产品简介

OpenCopilot 是一款 macOS 桌面级 AI 智能协驾工具。它在后台静默运行，通过**鼠标右键**唤醒悬浮卡片，让你在任何软件中随时调用 AI 能力——无需切换窗口、无需复制粘贴、不打断心流。

**一句话总结**：在任何软件里，选中文字、双击右键、AI 就来了。

---

## 二、系统要求与权限设置

### 2.1 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS 15 (Sequoia) 或更高 |
| Python | 3.10 ~ 3.12（勿使用 3.13+） |
| 依赖 | `pip install -r requirements.txt` |

### 2.2 授予权限（一次性，首次运行前完成）

打开 **系统设置 → 隐私与安全性**，为你的终端程序（Terminal.app 或 iTerm2）授予：

| 权限 | 用途 |
|------|------|
| **辅助功能（Accessibility）** | 监听全局鼠标事件（双击/三击右键） |
| **屏幕录制** | 全局光标特效渲染 |

> 如果没有授予辅助功能权限，OpenCopilot 将无法检测鼠标双击/三击事件。

## 三、安装与首次启动

### 3.1 克隆项目

```bash
git clone https://github.com/Walter1218/OpenCopilot.git
cd OpenCopilot
pip install -r requirements.txt
```

### 3.2 注册守护进程（一次性，实现开机自启）

OpenCopilot 有两个后台服务需要常驻运行：

| 服务 | 端口 | 注册命令 |
|------|------|----------|
| Agent 智能体 | 18888 | `bash scripts/install_daemon.sh` |
| Broker 特权代理 | 18889 | `bash scripts/install_broker_daemon.sh` ⚠️ 需在**原生终端**运行 |

> ⚠️ Broker 安装脚本**必须在 macOS 原生 Terminal.app 中运行**，不要在 IDE 内置终端执行，否则会继承沙盒限制。

### 3.3 启动 UI

```bash
bash scripts/start_ui.sh
```

启动后，你会在屏幕角落看到 OpenCopilot 的彩色光标特效。如果标题栏出现 🟢 绿色状态点，说明一切就绪。

## 四、核心功能与使用方法

### 4.1 快捷悬浮卡片 — 双击右键

**这是最常用的功能。操作只需两步：**

1. **在任何软件中高亮选中一段文字** (系统会自动通过底层 AXUIElement 无感读取，无需按 Cmd+C 复制)
2. **双击鼠标右键**

一张半透明的悬浮卡片会在鼠标旁边弹出，自动带入你刚才选中的文字。选择一个快捷指令，AI 就会开始分析。

#### 快捷指令按钮

| 按钮 | 功能 | 适用场景 |
|------|------|----------|
| ✨ 自动 | AI 自动判断内容并给出分析 | 不确定用途时使用 |
| 🌐 翻译 | 翻译为中文 | 阅读英文文档/网页 |
| 💻 代码解析 | 以架构师视角解析代码 | 读代码、Code Review |
| ✍️ 润色 | 优化文字表达 | 写邮件、报告、文案 |
| 👁️ 视觉分析前台 | 瞬间截取最前台窗口图像进行多模态分析 | 遇到无法提取文本的图片、界面元素或图表时使用 |

#### 自定义指令修改

拖拽/粘贴文本后，卡片会出现**指令输入栏**。在这里输入你的修改要求（如"将这段改成被动语态"），然后按 Enter 或点击 ▶ 按钮提交。AI 会严格按指令修改文本并只输出修改结果。

> 也可以直接点快捷指令按钮跳过输入栏，使用预设指令。

#### 结果操作按钮

AI 回复完成后，卡片底部会出现：

| 按钮 | 功能 |
|------|------|
| 📝 回写到 IDE | 将修改结果直接写回 IDE 中的原文件（需 IDE 插件） |
| 📋 复制结果 | 将结果复制到剪贴板 |
| 📋 粘贴剪贴板 | 将 Cmd+C 复制的文本粘贴到卡片（替代拖拽） |

#### 卡片交互

| 操作 | 效果 |
|------|------|
| 拖拽角落 | 缩放卡片大小 |
| 拖拽标题栏 | 移动卡片位置 |
| 点击 ✕ 按钮 | 关闭卡片 |
| 📋 粘贴剪贴板按钮 | 将 Cmd+C 复制的文本粘贴到卡片（适用于无法拖拽的场景，如 IDE 编辑区） |
| 卡片自动适配屏幕边缘 | 不会超出显示器边界 |

### 4.2 任务工作台 — 三击右键

**三击鼠标右键**会唤出一个更大的工作台窗口。在这里你可以：

- **设定任务背景**：在工作台顶部输入当前任务（例如"我正在撰写 Q3 产品报告，请帮我审阅逻辑一致性"），AI 在所有后续请求中都会记住这个背景。
- **独立深度对话**：在工作台中与 AI 进行不被打断的多轮对话，不依赖拖拽文本。

> 工作台设定的任务背景会自动注入到所有快捷卡片的 AI 请求中，实现"定义任务 → 划词执行"的聚焦工作流。

### 4.3 文档全文联动修订 — 📝 全文修订

**当你需要修改某篇文档中的一段文字，又担心改动后其他地方出现矛盾时，使用此功能。**

#### 操作步骤

1. **在 IDE 中打开文档**（.md / .txt / .py 等文本文件，确保 asu-ide-extension 插件已安装）
2. 双击右键唤出卡片
3. 点击 **「📝 全文修订」** 按钮 → 按钮变绿，系统自动读取文档全文
4. 在文档中**选中要修改的文字** → **拖拽到卡片**
5. AI 会输出三段结果：

```
📝 修订后文本        → 修改后的文字
🔍 联动影响分析      → 文档中其他需要同步调整的位置
💡 修订说明          → 本次修改的逻辑说明
```

#### 使用 Word / PowerPoint 文档

如果 IDE 中无法打开 .docx 或 .pptx 文件：

1. 点「📝 全文修订」→ 系统弹出文件选择框
2. 选择你的 .docx / .pptx 文件 → 按钮变绿
3. 在 Word/PPT 中选中要修改的文字 → 拖拽到卡片

> 系统通过 Privileged Broker 自动解析 .docx/.pptx 为纯文本进行全文交叉扫描。

#### 示例效果

```
原文：当前研发团队总人数 35 人。Q3 有 2 位同事离职。

用户拖拽修改 Q3 离职人数描述 → AI 输出：

📝 修订后文本：Q3 末有 2 位同事离职，预计 Q4 补充 3 人，届时 36 人

🔍 联动影响分析：
   ⚠️ "当前研发团队总人数 35 人" → 建议改为 "Q3 末在册 35 人"
   原因：与修订后的"Q3 末"时间节点保持一致，"当前"存在歧义
```

### 4.4 IDE 全文读取与选区修改

用于将整个代码文件的内容投喂给 AI 进行分析，或在 IDE 中选中代码片段进行修改。

#### 前置条件

安装 OpenCopilot IDE 伴生插件（`asu-ide-extension/` 目录下的 .vsix 文件）到 VSCode / Trae / Cursor 中。

#### 使用步骤

**全文读取分析：**

1. 在 IDE 中打开任意代码文件
2. 双击右键唤出卡片
3. 点击绿色 **「📥 极速读取当前 IDE 全文」** 按钮
4. 当前文件全部内容被读入卡片
5. 点击任意快捷指令进行分析（推荐 💻 代码解析）

**选中代码修改：**

1. 在 IDE 中**选中要修改的代码片段**
2. 双击右键唤出卡片 → 点击 **「📥 极速读取当前 IDE 全文」**
3. 在指令输入栏输入修改要求（如"将此函数改为异步"）
4. 按 Enter 提交 → AI 输出修改后的代码
5. 点击 **「📝 回写到 IDE」** → 修改结果自动写回 IDE 原位替换

### 4.5 浏览器网页读取

无需手动复制粘贴，一键获取当前浏览器页面全文。

#### 前置条件

1. 确保 Broker 守护进程已运行：`bash scripts/install_broker_daemon.sh`
2. 浏览器需开启 **「允许 Apple 事件执行 JavaScript」**（Chrome/Safari 开发者菜单中）

#### 使用步骤

1. 在浏览器中打开目标网页
2. 双击右键唤出卡片
3. 如果检测到浏览器，卡片会自动显示橙色 **「🌐 一键读取当前网页全文」** 按钮
4. 点击按钮 → 页面全文自动注入卡片
5. 选择快捷指令进行分析

> 支持浏览器：Chrome / Safari / Brave / Edge / Arc

### 4.6 拖拽投喂（通用兜底方案）

**最通用的输入方式，适用于任何软件。**

1. 在任意软件中**选中文字**
2. 双击右键唤出卡片
3. 将高亮文字**直接拖入卡片**
4. 点击快捷指令

> 拖拽使用系统底层 MIME 传输，不会触发 Cmd+C 导致的焦点丢失问题。100% 安全。

### 4.8 角色工坊 (Persona Workshop)

点击悬浮卡片标题栏右上角的 **「🎭」** 图标，即可打开角色工坊。
在这里你可以：
- 增删改查存放于 `personas/` 目录下的自定义 Prompt。
- 即时编辑生效，无需重启软件。
- 根据自己的工作习惯定制特定的 AI 角色（如：前端专家、法务顾问、小红书文案等）。

### 4.9 连续对话

点击卡片顶部的 **「💬 连续对话」** 标签页，可以进入多轮对话模式。

- 输入文字 → 回车发送
- AI 记住整个对话历史
- 适合需要多轮追问的场景（如逐步深入分析一个问题）

### 4.10 设置面板

双击右键唤出卡片后，点击标题栏可以唤出设置面板（或从菜单栏触发）：

| 设置项 | 说明 |
|--------|------|
| 云端 LLM (MiniMax) | 默认引擎，开箱即用，需配置 API Key |
| 本地 LLM (Ollama/vLLM) | 填写 API Base URL（如 `http://localhost:11434/v1`）和模型名，支持离线使用 |
| 探测模型 | 自动扫描本地模型列表 |

## 五、常用管理命令

| 操作 | 命令 |
|------|------|
| 启动 UI | `bash scripts/start_ui.sh` |
| 安装 Agent 守护进程 | `bash scripts/install_daemon.sh` |
| 安装 Broker 守护进程 | `bash scripts/install_broker_daemon.sh` |
| 卸载 Agent 守护进程 | `bash scripts/uninstall_daemon.sh` |
| 卸载 Broker 守护进程 | `bash scripts/uninstall_broker_daemon.sh` |

| 查看 Broker 实时日志 | `tail -f ~/Library/Logs/ASU/broker_out.log` |
| 检查 Agent 是否在线 | `curl http://127.0.0.1:18888/health` |
| 检查 Broker 是否在线 | `curl -H "Authorization: Bearer $(cat ~/.asu_broker_token)" http://127.0.0.1:18889/health` |
| 手动启动 Agent（调试） | `python asu_custom_agent.py` |
| 手动启动 Broker（调试） | `cd asu_broker && python run.py` |

## 七、常见问题

### Q1：双击右键没反应？
→ 检查 **系统设置 → 隐私与安全性 → 辅助功能**，确认终端程序已被授权。

### Q2：按钮显示红色/"OpenCopilot 核心守护服务未启动"？
→ Agent 守护进程未运行。执行 `bash scripts/install_daemon.sh` 安装并启动。

### Q3：浏览器读取按钮不出现？
→ 确保 Broker 已安装且在**原生终端**中运行。执行 `bash scripts/install_broker_daemon.sh`。

### Q4：IDE 读取失败、提示"找不到插件端口"？
→ 确保 IDE 伴生插件已安装并在当前 IDE 窗口中激活。插件会自动在 `/tmp/asu_ide_port.txt` 写入端口号。

### Q5：.docx/.pptx 文件解析失败？
→ 确保已安装依赖：`pip install python-docx python-pptx`。确保 Broker 服务在线。
