# V5 — 事实锚点硬约束

> 迭代时间: 2026-06-11
> 基于: V4 baseline

## 本轮改动

**只改一类策略**：在 ppt_editor system prompt 核心原则区域追加 3 条事实忠实性硬约束（规则 8/9/10）。

### 新增规则

- 规则 8: **事实锚点不可丢失** — 改写/润色/压缩时每条输出必须保留至少一个事实锚点（数字/金额/百分比/时间/专有名词/英文缩写），禁止用"显著增长""大幅提升"等模糊表达替代具体数值
- 规则 9: **计划与结果严格区分** — 不得将"计划/预计/目标/拟"等未来态改写为"已完成/已实现/已达到"等过去态
- 规则 10: **风险与负面信息不可弱化** — 风险/限制/错误率/不确定性表达必须显式保留，不允许用积极措辞替换或省略

### 未改动部分

- 原有规则 1-7 保持不变
- 修改模式说明保持不变
- render_prompt_generator 保持不变（本轮只改 system prompt）

## 预期提升

- 数字/金额/百分比保留率上升
- 计划态 vs 完成态混淆减少
- 风险/负面信息被弱化的概率降低
- fr_001/002/004/007/008/009/010/011 等 case 表现改善

## 潜在风险

- 新增 3 条规则增加 prompt 长度，可能轻微影响延迟
- 规则过多可能导致模型注意力分散，需要观察是否影响其他维度
