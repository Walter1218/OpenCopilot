# V6 — 结构保持与条目粒度约束

> 迭代时间: 2026-06-11
> 基于: V5 事实锚点硬约束

## 本轮改动

**只改一类策略**：在 V5 基础上新增「结构保持」硬约束（规则 11/12）。

### 新增规则

- 规则 11: **条目数量和顺序保持不变** — 润色/改写时，原有 items 条目数量不得增减，顺序不得调整，不允许多条合并为一条，也不允许一条拆分为多条
- 规则 12: **区间不可单点化** — 原文中的价格区间、时间区间、百分比区间，不得被压缩为单一数值

### render_prompt_generator 增强

- 在 text 类型示例中新增「忠实改写」示例，展示保留事实锚点的 before/after 对照

### 未改动部分

- 原有规则 1-10 保持不变
- 修改模式说明保持不变

## 预期提升

- 条目合并/拆分问题减少（fr_003/fr_005）
- 区间数据被单点化的问题减少（fr_008）
- 结构保持度维度整体改善

## 潜在风险

- 规则 11 可能与"添加要点"操作产生语义冲突，需观察 add_item 类 case
- 规则 12 对非改写类指令（如图表转换）无影响，但增加 prompt 长度
